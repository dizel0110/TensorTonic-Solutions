## 📋 Требования задачи

**Задача:**
- Реализовать один шаг оптимизатора RMSProp (Root Mean Square Propagation)
- Даны: параметры `w`, градиенты `g`, накопитель квадратов градиентов `s`
- Вернуть: обновлённые параметры и накопитель

**Формулы обновления:**

```
Step 1: Update Running Average
s_t = β · s_{t-1} + (1 - β) · g_t²

Step 2: Parameter Update
w_t = w_{t-1} - lr · g_t / (√s_t + ε)
```

**Где:**
- `w` — параметры (любая форма)
- `g` — градиенты (та же форма, что и `w`)
- `s` — накопитель квадратов градиентов (та же форма)
- `lr` (η) — learning rate
- `β` — коэффициент затухания скользящего среднего
- `ε` — константа для численной стабильности

**Ограничения:**
- Вернуть кортеж `(new_w, new_s)` с той же формой, что и входы
- Использовать точные формулы выше
- Векторизованная реализация (без циклов)
- Поддержка любой формы массива (1D, 2D, и т.д.)
- Размерность параметров: 1 ≤ D ≤ 10⁵
- Learning rate `lr > 0`
- Decay factor: `0 < β < 1`
- Только NumPy

**Гиперпараметры по умолчанию:**
- `lr = 0.001`
- `beta = 0.9`
- `eps = 1e-8`

## 💡 Подсказки из задачи

1. **Конвертация в NumPy:** Сначала конвертировать входы в NumPy массивы. Обновить `s` используя формулу экспоненциального скользящего среднего, затем использовать для обновления `w`.

2. **Поэлементные операции:** Использовать `g * g` для квадратов градиентов, `np.sqrt()` для квадратного корня, стандартные арифметические операторы.

---

## 🔧 Решение

**Ключевые моменты реализации:**

```python
def rmsprop_step(w, g, s, lr=0.001, beta=0.9, eps=1e-8):
    # 1. Конвертация входов в NumPy array для векторизации
    w = np.asarray(w, dtype=float)
    g = np.asarray(g, dtype=float)
    s = np.asarray(s, dtype=float)

    # Step 1: Update running average of squared gradients
    # s_t = β * s_{t-1} + (1 - β) * g_t²
    new_s = beta * s + (1 - beta) * (g ** 2)

    # Step 2: Parameter update
    # w_t = w_{t-1} - lr * g_t / (√s_t + ε)
    new_w = w - lr * g / (np.sqrt(new_s) + eps)

    # Гарантируем возврат np.ndarray (для скаляров это 0-d array)
    return (np.asarray(new_w), np.asarray(new_s))
```

**Почему это работает:**

1. **RMSProp идея:**
   - Адаптивный learning rate для каждого параметра
   - Большие градиенты → большое `s` → меньшее обновление
   - Малые градиенты → малое `s` → большее обновление

2. **Накопитель квадратов градиентов (`s`):**
   - Экспоненциальное скользящее среднее квадратов градиентов
   - `beta * s` — сохраняем историю
   - `(1 - beta) * g²` — добавляем текущий квадрат градиента
   - Оценивает дисперсию градиентов

3. **Адаптивное обновление:**
   - Деление на `√s` нормализует градиент
   - Параметры с большими градиентами обновляются медленнее
   - Параметры с малыми градиентами обновляются быстрее

4. **Численная стабильность:**
   - `eps` в знаменателе предотвращает деление на ноль
   - Важно когда `s` близко к нулю

5. **Векторизация:**
   - `np.asarray()` работает со скалярами, списками, массивами любой размерности
   - Все операции поэлементные благодаря NumPy broadcasting

## 📊 Пример работы

```python
import numpy as np
from rmsprop_optimizer import rmsprop_step

# Пример 1: Базовое использование
w = np.array([1.0, 2.0])
g = np.array([0.2, -0.4])
s = np.array([0.0, 0.0])
lr = 0.1

new_w, new_s = rmsprop_step(w, g, s, lr=lr)
print(f"new_w: {new_w}")  # [0.684, 2.316]
print(f"new_s: {new_s}")  # [0.004, 0.016]

# Пример 2: Нулевой градиент
w = np.array([5.0])
g = np.array([0.0])
s = np.array([0.1])

new_w, new_s = rmsprop_step(w, g, s, lr=0.1, beta=0.9)
print(f"new_w: {new_w}")  # [5.0] - не меняется
print(f"new_s: {new_s}")  # [0.09] - decays: 0.9 * 0.1 = 0.09

# Пример 3: Адаптивный learning rate
w = np.array([0.0, 0.0])
g = np.array([1.0, 1.0])
s = np.array([0.01, 1.0])  # Разные s

new_w, _ = rmsprop_step(w, g, s, lr=0.1)
print(f"new_w: {new_w}")
# Первый элемент обновится больше (s=0.01 → √s=0.1)
# Второй элемент обновится меньше (s=1.0 → √s=1.0)
```

## 📝 Примечания

**RMSProp vs SGD:**

| Характеристика | SGD | RMSProp |
|---------------|-----|---------|
| Learning rate | Единый для всех | Адаптивный на параметр |
| Градиенты |直接使用 | Нормализуются через √s |
| Для нестационарных | Может осциллировать | Лучше справляется |
| RNN/LSTM | Проблемы | Хорошо работает |

**RMSProp vs Adam:**

| Характеристика | RMSProp | Adam |
|---------------|---------|------|
| Моменты | Только 2-й (s) | 1-й (m) и 2-й (v) |
| Bias correction | Нет | Да |
| Память | Меньше | Больше |
| Сходимость | Хорошая | Обычно лучше |

**Типичные значения гиперпараметров:**
- `lr = 0.001` (начальное, может требовать tuning)
- `beta = 0.9` (обычно не меняется)
- `eps = 1e-8` (защита от деления на ноль)

**Когда использовать RMSProp:**
- ✅ Нестационарные цели
- ✅ RNN/LSTM
- ✅ Шумные градиенты
- ✅ Требуется адаптивный lr

**Когда может быть лучше Adam:**
- ✅ Глубокие сети общего назначения
- ✅ Требуется быстрая сходимость
- ✅ Есть вычислительные ресурсы
